Parser Project

This project was done in the CS 441 class at UMKC. In this project, a parser was written which would parse a simplified version of 70s-era BASIC code. The grammar that was parsed is shown below. The parser was written in the Racket programming langauge using the DrRacket IDE. The parser was developed using a technique known as top-down parsing where the program scans the code from top to bottom and tries to match the given code with the rules listed in the grammar.

The grammar that was parsed which lists all the rules the code must follow for the file to be succesfully parsed.

This is one of the sample files that would be parsed.

If a file had a syntax error, the program would raise an error.
Otherwise the program would print out "Accept".
